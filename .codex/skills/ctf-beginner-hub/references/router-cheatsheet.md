# 模式选择速查表

## 先选哪种模式？

### 直接走自动分流
如果你已经有这些东西里的任意一种：
- 附件
- URL
- IP:PORT
- 源码
- 二进制
- PCAP / 内存 / 磁盘镜像

一句话：**有材料，先自动分流。**

### 先走头脑风暴
如果你是下面这种情况：
- 只有一句题面
- 术语很多，看不懂
- 不确定目标到底是找 flag、打漏洞还是还原逻辑
- 想先理解，再动手

一句话：**没想清楚，先头脑风暴。**

## 自动分流时怎么判断？

- HTTP 页面 / API / 登录框 / 上传功能 -> `ctf-web`
- ELF / EXE / APK / WASM / 固件 -> 优先 `ctf-reverse`
- 二进制服务 + 明显要打崩 / 劫持执行流 -> `ctf-pwn`
- 一堆数字 / 模运算 / RSA / AES / 随机数 -> `ctf-crypto`
- PCAP / 内存 / 日志 / 磁盘 / 隐写 -> `ctf-forensics`
- 社交账号 / 地理位置 / 公开资料 -> `ctf-osint`
- 混淆脚本 / 样本 / C2 / beacon -> `ctf-malware`
- pyjail / bash jail / 编码 / 游戏 / 奇怪规则题 -> `ctf-misc`
- AI 模型 / prompt / jailbreak / 对抗样本 -> `ctf-ai-ml`

## 不确定时的推荐输出

不要硬判，先输出：
- 第一候选
- 第二候选
- 还缺什么证据
- 下一步最小验证动作
